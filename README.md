# Найденные и исправленные проблемы

---

## 1. Нарушение принципа разделения интерфейсов (ISP — SOLID)

**Было:** Абстрактный класс `CurrencyConverter` объявлял 4 абстрактных метода (`convert_usd_to_eur`, `convert_usd_to_gbp`, `convert_usd_to_rub`, `convert_usd_to_cny`), хотя каждый конкретный конвертер реализует только один из них.

**Стало:** Единственный абстрактный метод `convert(amount: float) -> float`. Каждый класс реализует ровно то, что ему нужно.

---

## 2. Нарушение принципа подстановки Лисков (LSP — SOLID)

**Было:** Каждый конвертер вынужден был реализовывать все 4 метода. Три "лишних" метода просто вызывали `print('This is not ... converter')` и возвращали `None` вместо числа — контракт нарушен, вызывающий код получал `None` там, где ожидалось `float`.

**Стало:** Устранено вместе с ISP — у каждого класса один метод `convert()`, который всегда возвращает `float`.

---

## 3. Нарушение принципа единственной ответственности (SRP — SOLID)

**Было:** Каждый конвертер совмещал в себе:
- логику HTTP-запроса к API
- логику повторных попыток (retry)
- логику кэширования (в `UsdCnyConverter`)
- настройку логирования
- собственно конвертацию валюты

**Стало:** Ответственности разделены:
- `rate_fetcher.py` — получение курсов с API (с retry и кэшированием)
- `base_usd_converter.py` — базовый класс с вызовом `fetch_rates()`
- Каждый конвертер — только знает свой код валюты и реализует `convert()`

---

## 4. Нарушение принципа DRY (Don't Repeat Yourself)

**Было:**
- `usd_eur_converter.py` и `usd_gbp_converter.py` были идентичны на 95% — одинаковый `__init__`, `_setup_logger()`, `get_rates()` с retry, заглушки для других методов
- `_setup_logger()` дублировался в EUR и GBP конвертерах
- `get_rates()` с retry-логикой дублировался в EUR и GBP конвертерах
- В `main.py` один и тот же паттерн `converter = ...; print(...)` повторялся 4 раза

**Стало:**
- `fetch_rates()` с retry — один раз в `rate_fetcher.py`
- `BaseUsdConverter` содержит общую логику; EUR/GBP/RUB конвертеры — по 3 строки каждый
- `main.py` использует цикл по списку конвертеров

---

## 5. Баг: накопление обработчиков логгера

**Было:** Метод `_setup_logger()` в EUR и GBP конвертерах каждый раз добавлял новый `StreamHandler` к модульному логгеру через `logger.addHandler(ch)`. При создании нескольких экземпляров каждое сообщение дублировалось в консоли.

**Стало:** Логирование настраивается один раз через `logging.basicConfig()` в `main.py`. В остальных модулях используется `logging.getLogger(__name__)` без ручного добавления обработчиков.

---

## 6. Баг: обращение к `self.rates` без проверки на `None`

**Было:** `get_rates()` мог вернуть `None` (при ошибке сети), но все методы конвертации сразу обращались к `self.rates['EUR']` и т.д., что приводило к `TypeError` в рантайме.

**Стало:** В `BaseUsdConverter.convert()` и `UsdCnyConverter.convert()` добавлена явная проверка: если `self.rates is None` — бросается `RuntimeError` с понятным сообщением.

---

## 7. Несогласованная обработка ошибок

**Было:** EUR/GBP конвертеры использовали `logging`, а `UsdCnyConverter` использовал `print()` для вывода ошибок и предупреждений.

**Стало:** Весь код использует `logging` единообразно, включая `rate_fetcher.py`, который обслуживает CNY конвертер.

---

## 8. Нарушение YAGNI — неиспользуемый импорт

**Было:** `import asyncio` в `main.py` — модуль нигде не использовался.

**Стало:** Импорт удалён.

---

## 9. Wildcard-импорт (нарушение чистого кода)

**Было:** `from converters import *` в `main.py` — неявно импортирует неизвестный набор имён, загрязняет пространство имён.

**Стало:** Явный импорт: `from converters import UsdEurConverter, UsdGbpConverter, UsdRubConverter, UsdCnyConverter`.

---

## 10. Отсутствие валидации пользовательского ввода

**Было:** `int(input(...))` — при нечисловом вводе программа падала с необработанным `ValueError`.

**Стало:** Ввод обёрнут в `try/except ValueError` с понятным сообщением об ошибке. Тип изменён на `float` для поддержки дробных сумм.

---

## 11. Отсутствие аннотаций типов (нарушение чистого кода)

**Было:** Ни один метод не имел аннотаций типов.

**Стало:** Все публичные методы и функции аннотированы (`float`, `Optional[dict]`, `None`, `str`, `int`).
